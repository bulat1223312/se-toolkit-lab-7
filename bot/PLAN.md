# План разработки Telegram-бота

## Обзор
Мы создадим Telegram-бота, который будет взаимодействовать с существующим LMS API и LLM (например, OpenAI) для ответов на вопросы о лабораторных работах и оценках. Бот будет построен так, чтобы быть тестируемым: обработчики команд — это чистые функции, которые принимают входные данные и возвращают текст, не завися от Telegram.

## Задачи

### 1. Создание каркаса проекта (Задача 1)
- Создать `pyproject.toml` с зависимостями: `python-telegram-bot`, `requests`, `python-dotenv`.
- Написать `config.py` для загрузки переменных окружения (BOT_TOKEN, LMS_API_BASE_URL, LMS_API_KEY, LLM_API_KEY).
- Реализовать точку входа `bot.py` с двумя режимами:
  - Обычный режим: запуск Telegram-бота.
  - Тестовый режим (`--test`): вызов функций-обработчиков напрямую и вывод результата в stdout.
- Создать заглушки обработчиков в `handlers/` (например, `start`, `help`, `health`, `scores`, `labs`).
- Предоставить `.env.bot.example` с переменными-заглушками.

### 2. Интеграция с бэкендом (Задача 2)
- Создать `services/lms_client.py` для общения с LMS API (GET /health, GET /labs, GET /scores/<lab_id>).
- Реализовать обработчики, которые используют этот клиент и форматируют ответы в виде текста.

### 3. Маршрутизация намерений с LLM (Задача 3)
- Добавить LLM-клиент (`services/llm_client.py`) для классификации сообщений пользователя (не команд) по намерениям: оценки, лабораторные, неизвестно.
- Реализовать fallback-обработчик, который отправляет сообщение в LLM для маршрутизации и возвращает соответствующий ответ.

### 4. Развёртывание и тестирование
- Убедиться, что все обработчики работают в тестовом режиме (`--test`).
- Развернуть бота на виртуальной машине: получить изменения, установить зависимости через `uv`, запустить бота в фоне.
- Проверить работу в Telegram.
